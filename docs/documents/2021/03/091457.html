项目初始化

网络问题

Github访问有问题的同学，可以参看码云上同步的项目。其操作步骤与Github基本一致。

Git

  1. fork项目

  2. 克隆项目到本地

  3. 项目地址

    • Github
    • Gitee

  4. 克隆项目代码

    注意，下面{yourname in github}需要替换成自己在Github或Gitee中的用户名，如git clone git@github.com:john/p000025.git或git clone git@gitee.com:jane/p000001.git

    • Github

      mkdir ~/01factory
cd ~/01factory
git clone git@github.com:{yourname in github}/p000025.git
cd ./p000025/
yarn

    • Gitee

      mkdir ~/01factory
cd ~/01factory
git clone git@gitee.com:{yourname in gitee}/p000001.git
cd ./p000001/
yarn

实操视频

Local

这种方式无法提交代码，但可以本地运行

  • Github

    mkdir ~/01factory
cd ~/01factory
wget https://github.com/mm-works/p000025/archive/main.zip
unzip ./main.zip
mv ./p000025-main ./p000025
rm -f ./main.zip
cd ./p000025/
yarn

  • Gitee

    mkdir ~/01factory
cd ~/01factory
wget https://gitee.com/dfactory01/p000001/repository/archive/main.zip
unzip ./main.zip
rm -f ./main.zip
cd ./p000001/
yarn

这种方式简单，可以在学习时使用，但是真正开发时是不会使用的，我们的软件项目的代码必须要有一个代码版本管理工具，即使这个项目只有一个人在开发，即使这个项目不需要放在像github这样的软件仓库。
